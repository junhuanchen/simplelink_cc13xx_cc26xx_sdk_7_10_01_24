#define ATTRID_OTA_UPGRADE_DOWNLOADED_ZIG_BEE_STACK_VERSION   0x0005

The ZigBee stack version of the downloaded image on additional memory space on the device.

§ ATTRID_OTA_UPGRADE_IMAGE_STAMP

#define ATTRID_OTA_UPGRADE_IMAGE_STAMP   0x000A

This attribute acts as a second verification to identify the image in the case that sometimes developers of the application have forgotten to increase the firmware version attribute.

§ ATTRID_OTA_UPGRADE_IMAGE_TYPE_ID

#define ATTRID_OTA_UPGRADE_IMAGE_TYPE_ID   0x0008

This attribute SHALL indicate the image type identifier of the file that the client is currently downloading, or a file that has been completely downloaded but not upgraded to yet.

§ ATTRID_OTA_UPGRADE_IMAGE_UPGRADE_STATUS

#define ATTRID_OTA_UPGRADE_IMAGE_UPGRADE_STATUS   0x0006

The upgrade status of the client device. The status indicates where the client device is at in terms of the download and upgrade process.

§ ATTRID_OTA_UPGRADE_MANUFACTURER_ID

#define ATTRID_OTA_UPGRADE_MANUFACTURER_ID   0x0007

This attribute SHALL reflect the ZigBee assigned value for the manufacturer of the device.

§ ATTRID_OTA_UPGRADE_MINIMUM_BLOCK_PERIOD

#define ATTRID_OTA_UPGRADE_MINIMUM_BLOCK_PERIOD   0x0009

This attribute acts as a rate limiting feature for the server to slow down the client download and prevent saturating the network with block requests.

§ ATTRID_OTA_UPGRADE_UPGRADE_SERVER_ID

#define ATTRID_OTA_UPGRADE_UPGRADE_SERVER_ID   0x0000

The attribute is used to store the IEEE address of the upgrade server resulted from the discovery of the upgrade server's identity

§ COMMAND_OTA_UPGRADE_IMAGE_BLOCK_REQUEST

#define COMMAND_OTA_UPGRADE_IMAGE_BLOCK_REQUEST   0x03

The client device requests the image data at its leisure by sending Image Block Request command to the upgrade server.

§ COMMAND_OTA_UPGRADE_IMAGE_BLOCK_RESPONSE

#define COMMAND_OTA_UPGRADE_IMAGE_BLOCK_RESPONSE   0x05

Upon receipt of an Image Block Request command the server SHALL generate an Image Block Response.

§ COMMAND_OTA_UPGRADE_IMAGE_NOTIFY

#define COMMAND_OTA_UPGRADE_IMAGE_NOTIFY   0x00

The purpose of sending Image Notify command is so the server has a way to notify client devices of when the OTA upgrade images are available for them.

§ COMMAND_OTA_UPGRADE_IMAGE_PAGE_REQUEST

#define COMMAND_OTA_UPGRADE_IMAGE_PAGE_REQUEST   0x04

The support for the command is optional. The client device MAY choose to request OTA upgrade data in one page size at a time from upgrade server.

§ COMMAND_OTA_UPGRADE_QUERY_DEVICE_SPECIFIC_FILE_REQUEST

#define COMMAND_OTA_UPGRADE_QUERY_DEVICE_SPECIFIC_FILE_REQUEST   0x08

Client devices SHALL send a Query Device Specific File Request command to the server to request for a file that is specific and unique to it.

§ COMMAND_OTA_UPGRADE_QUERY_DEVICE_SPECIFIC_FILE_RESPONSE

#define COMMAND_OTA_UPGRADE_QUERY_DEVICE_SPECIFIC_FILE_RESPONSE   0x09

The server sends Query Device Specific File Response after receiving Query Device Specific File Request from a client.

§ COMMAND_OTA_UPGRADE_QUERY_NEXT_IMAGE_REQUEST

#define COMMAND_OTA_UPGRADE_QUERY_NEXT_IMAGE_REQUEST   0x01

Client devices SHALL send a Query Next Image Request command to the server to see if there is new OTA upgrade image available.

§ COMMAND_OTA_UPGRADE_QUERY_NEXT_IMAGE_RESPONSE

#define COMMAND_OTA_UPGRADE_QUERY_NEXT_IMAGE_RESPONSE   0x02

The upgrade server sends a Query Next Image Response with one of the following status: SUCCESS, NO_IMAGE_AVAILABLE or NOT_AUTHORIZED. When a SUCCESS status is sent, it is considered to be the explicit authorization to a device by the upgrade server that the device MAY upgrade to a specific software image.

§ COMMAND_OTA_UPGRADE_UPGRADE_END_REQUEST

#define COMMAND_OTA_UPGRADE_UPGRADE_END_REQUEST   0x06

Upon reception all the image data, the client SHOULD verify the image to ensure its integrity and validity.

§ COMMAND_OTA_UPGRADE_UPGRADE_END_RESPONSE

#define COMMAND_OTA_UPGRADE_UPGRADE_END_RESPONSE   0x07

When an upgrade server receives an Upgrade End Request command with a status of INVALID_IMAGE, REQUIRE_MORE_IMAGE, or ABORT, no additional processing SHALL be done in its part. If the upgrade server receives an Upgrade End Request command with a status of SUCCESS, it SHALL generate an Upgrade End Response 17245 with the manufacturer code and image type received in the Upgrade End Request along with the times indicating when the device SHOULD upgrade to the new image.

§ OTA_MAX_MTU

#define OTA_MAX_MTU   32

The value indicates the largest possible length of data (in bytes) that the client can receive at once.
