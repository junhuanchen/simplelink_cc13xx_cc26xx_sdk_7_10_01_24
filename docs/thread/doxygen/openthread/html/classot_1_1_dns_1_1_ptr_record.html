This method parses and reads the PTR name from a message.

This method also verifies that the PTR record is well-formed (e.g., the record data length GetLength() matches the PTR encoded name).

Parameters
[in]aMessageThe message to read from. aMessage.GetOffset() MUST point to the start of DNS header.
[in,out]aOffsetOn input, the offset in aMessage to start of PTR name field. On exit when successfully read, aOffset is updated to point to the byte after the entire PTR record (skipping over the record).
[out]aNameBufferA pointer to a char array to output the read name as a null-terminated C string (MUST NOT be nullptr).
[in]aNameBufferSizeThe size of aNameBuffer.
Return values
kErrorNoneThe PTR name was read successfully. aOffset and aNameBuffer are updated.
kErrorParseThe PTR record in aMessage could not be parsed (invalid format).
kErrorNoBufsName could not fit in aNameBufferSize chars.

This method parses and reads the PTR name from a message.

This method also verifies that the PTR record is well-formed (e.g., the record data length GetLength() matches the PTR encoded name).

Unlike the previous method which reads the entire PTR name into a single char buffer, this method reads the first label separately and into a different buffer aLabelBuffer and the rest of the name into aNameBuffer. The aNameBuffer can be set to nullptr if the caller is only interested in the first label. This method is intended for "Service Instance Name" where first label (<Instance> portion) can be a user-friendly string and can contain dot character.

Parameters
[in]aMessageThe message to read from. aMessage.GetOffset() MUST point to the start of DNS header.
[in,out]aOffsetOn input, the offset in aMessage to the start of PTR name field. On exit, when successfully read, aOffset is updated to point to the byte after the entire PTR record (skipping over the record).
[out]aLabelBufferA pointer to a char array to output the first label as a null-terminated C string (MUST NOT be nullptr).
[in]aLabelBufferSizeThe size of aLabelBuffer.
[out]aNameBufferA pointer to a char array to output the rest of name (after first label). Can be nullptr if caller is only interested in the first label.
[in]aNameBufferSizeThe size of aNameBuffer.
Return values
kErrorNoneThe PTR name was read successfully. aOffset, and are updated.
kErrorParseThe PTR record in aMessage could not be parsed (invalid format).
kErrorNoBufsEither label or name could not fit in the related char buffers.
