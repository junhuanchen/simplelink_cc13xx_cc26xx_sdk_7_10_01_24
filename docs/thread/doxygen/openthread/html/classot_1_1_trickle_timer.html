§ IndicateConsistent()

void IndicateConsistent ( void  )

This method indicates to the trickle timer a 'consistent' event.

The 'consistent' events are used to control suppression behavior. The trickle timer keeps track of the number of 'consistent' events in each interval. The timer handler is invoked only if the number of consistent events received in the interval is less than the redundancy constant.

References TrickleTimer::kInfiniteRedundancyConstant, and TrickleTimer::mCounter.

§ IndicateInconsistent()

void IndicateInconsistent ( void  )

This method indicates to the trickle timer an 'inconsistent' event.

Receiving an 'inconsistent' event causes the trickle timer to reset (i.e., start with interval set to the min value) unless the current interval being used is already equal to the min interval.
