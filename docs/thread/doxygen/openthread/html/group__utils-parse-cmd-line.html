§ ParseAsHexString() [1/3]

otError ParseAsHexString ( const char *  aString,
uint8_t *  aBuffer,
uint16_t  aSize 
)

This function parses a hex string into a byte array of fixed expected size.

This function returns kErrorNone only when the hex string contains exactly aSize bytes (after parsing). If there are fewer or more bytes in hex string that aSize, the parsed bytes (up to aSize) are copied into the aBuffer and kErrorInvalidArgs is returned.

This function correctly handles hex strings with even or odd length. For example, "AABBCCDD" (with even length) is parsed as {0xaa, 0xbb, 0xcc, 0xdd} and "123" (with odd length) is parsed as {0x01, 0x23}.

Parameters
[in]aStringThe string to parse.
[out]aBufferA pointer to a buffer to output the parsed byte sequence.
[in]aSizeThe expected size of byte sequence (number of bytes after parsing).
Return values
kErrorNoneThe string was parsed successfully.
kErrorInvalidArgsThe string does not contain valid hex bytes and/or not aSize bytes.

§ ParseAsHexStringSegment()

otError ParseAsHexStringSegment ( const char *&  aString,
uint16_t &  aSize,
uint8_t *  aBuffer 
)

This function parses a segment of a hex string up to a given size.

This function allows a longer hex string to be parsed and read in smaller segments into a given buffer. If the entire hex string bytes can fit in the given aBuffer with its aSize, they are copied into aBuffer and function returns kErrorNone. Otherwise, aSize bytes are read and copied and function returns kErrorPending to indicate that there are more bytes to parse. The aString is also updated to skip over the parsed segment.

This function correctly handles hex strings with even or odd length. For example, "AABBCCDD" (with even length) is parsed as {0xaa, 0xbb, 0xcc, 0xdd} and "123" (with odd length) is parsed as {0x01, 0x23}.

Parameters
[in,out]aStringA reference to string to parse. On successful parse, updated to skip parsed digits.
[in,out]aSizeOn entry indicates the segment size (number of bytes in aBuffer). On exit provides number of bytes parsed and copied into aBuffer.
[out]aBufferA pointer to a buffer to output the parsed byte sequence.
Return values
kErrorNoneThe string was parsed successfully to the end of string.
kErrorPedningThe string segment was parsed successfully, but there are additional bytes remaining to be parsed.
kErrorInvalidArgsThe string does not contain valid format hex digits.

§ ParseCmd() [1/2]

otError ParseCmd ( char *  aCommandString,
Arg  aArgs[],
uint8_t  aArgsMaxLength 
)

This function parses a given command line string and breaks it into an argument list.

This function may change the input aCommandString, it will put a '\0' by the end of each argument, and aArgs will point to the arguments in the input aCommandString. Backslash ('\') can be used to escape separators (' ', '', '', '
') and the backslash itself.

As the arguments are parsed, the aArgs array entries are populated. Any remaining aArgs entries in the array will be cleared and marked as empty. So the number of arguments can be determined by going through aArgs array entries till we get to an empty Arg (i.e., Arg::IsEmpty() returns `true).

This function ensures that the last entry in aArgs array is always used to indicate the end (always marked as empty), so the aArgs array should have one more entry than the desired max number of arguments.

Parameters
[in]aCommandStringA null-terminated input string.
[out]aArgsThe argument array.
[in]aArgsMaxLengthThe max length of aArgs array.
Return values
OT_ERROR_NONEThe command line parsed successfully and aArgs array is populated.
OT_ERROR_INVALID_ARGSToo many arguments in aCommandString and could not fit in aArgs array.
