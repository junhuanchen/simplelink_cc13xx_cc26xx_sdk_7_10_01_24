Detailed Description

This class represents a DNS name and implements helper methods for encoding/decoding of DNS Names.

§ AppendTo()

Error AppendTo ( Message &  aMessage) const

This method encodes and appends the name to a message.

If the name is empty (not specified), then root "." is appended to aMessage. If the name is from a C string then the string is checked and appended (similar to static AppendName(const char *aName, Message &) method). If the the name is from a message, then it is read from the message and appended to aMessage. Note that in this case independent of whether the name is compressed or not in its original message, the name is appended as full (uncompressed) in aMessage.

Parameters
[in]aMessageThe message to append to.
Return values
kErrorNoneSuccessfully encoded and appended the name to aMessage.
kErrorInvalidArgsName is not valid.
kErrorNoBufsInsufficient available buffers to grow the message.

This static method encodes and appends a single name label to a message.

The aLabel is assumed to contain a single name label as a C string (null-terminated). Unlike AppendMultipleLabels() which parses the label string and treats it as sequence of multiple (dot-separated) labels, this method always appends aLabel as a single whole label. This allows the label string to even contain dot '.' character, which, for example, is useful for "Service Instance Names" where <Instance> portion is a user-friendly name and can contain dot characters.

§ AppendMultipleLabels() [1/2]

Error AppendMultipleLabels ( const char *  aLabels,
Message &  aMessage 
)
static

This static method encodes and appends a sequence of name labels to a given message.

The aLabels must follow "<label1>.<label2>.<label3>", i.e., a sequence of labels separated by dot '.' char. E.g., "_http._tcp", "_http._tcp." (same as previous one), "host-1.test".

This method validates that the aLabels is a valid name format, i.e., no empty label, and labels are kMaxLabelLength (63) characters or less.

Note
This method NEVER adds a label terminator (empty label) to the message, even in the case where aLabels ends with a dot character, e.g., "host-1.test." is treated same as "host-1.test".
Parameters
[in]aLabelsA name label string. Can be nullptr (then treated as "").
[in]aMessageThe message to which to append the encoded name.
Return values
kErrorNoneSuccessfully encoded and appended the name label(s) to aMessage.
kErrorInvalidArgsName label aLabels is not valid.
kErrorNoBufsInsufficient available buffers to grow the message.

Error ReadLabel ( const Message &  aMessage,

char *  aLabelBuffer,

static

This static method reads a name label from a message.

This method can be used to read labels one by one in a name. After a successful label read, aOffset is updated to point to the start of the next label. When we reach the end of the name, kErrorNotFound is returned. This method handles compressed names which use pointer labels. So as the labels in a name are read, the aOffset may jump back in the message and at the end the aOffset does not necessarily point to the end of the original name field.

Unlike ReadName() which requires and verifies that the read label to contain no dot '.' character, this method allows the read label to include any character.

Parameters
[in]aMessageThe message to read the label from. aMessage.GetOffset() MUST point to the start of DNS header (this is used to handle compressed names).
[in,out]aOffsetOn input, the offset in aMessage pointing to the start of the label to read. On exit, when successfully read, aOffset is updated to point to the start of the next label.
[out]aLabelBufferA pointer to a char array to output the read label as a null-terminated C string.
[in,out]aLabelLengthOn input, the maximum number chars in aLabelBuffer array. On output, when label is successfully read, aLabelLength is updated to return the label's length (number of chars in the label string, excluding the null char).
Return values

This static method parses and compares a full name from a message with a given name.

This method checks whether the encoded name in a message matches a given name string (using case-insensitive comparison). It checks the name in the message in place and handles compressed names. If the name read from the message does not match aName, it returns kErrorNotFound. kErrorNone indicates that the name matches aName.

The aName must follow "<label1>.<label2>.<label3>", i.e., a sequence of labels separated by dot '.' char. E.g., "example.com", "example.com." (same as previous one), "local.", "default.service.arpa", "." or "" (root).

const Message &  aMessage2,

static

This static method parses and compares a full name from a message with a name from another message.

This method checks whether the encoded name in aMessage matches the name from aMessage2 (using case-insensitive comparison). It compares the names in both messages in place and handles compressed names. Note that this method works correctly even when the same message instance is used for both aMessage and aMessage2 (e.g., at different offsets).

Only the name in aMessage is fully parsed and checked for parse errors. This method assumes that the name in aMessage2 was previously parsed and validated before calling this method (if there is a parse error in aMessage2, it is treated as a name mismatch with aMessage).

If the name in aMessage can be parsed fully (independent of whether the name matches or not with the name from aMessage2), the aOffset is updated (note that aOffset2 for aMessage2 is not changed).

Parameters
[in]aMessageThe message to read the name from and compare. aMessage.GetOffset() MUST point to the start of DNS header (this is used to handle compressed names).
[in,out]aOffsetOn input, the offset in aMessage pointing to the start of the name field. On exit (when parsed successfully independent of whether the read name matches or not), aOffset is updated to point to the byte after the end of the name field.
[in]aMessage2The second message to read the name from and compare with name from aMessage. aMessage2.GetOffset() MUST point to the start of DNS header.
[in]aOffset2The offset in aMessage2 pointing to the start of the name field.
Return values
kErrorNoneThe name from aMessage matches the name from aMessage2. aOffset is updated.
kErrorNotFoundThe name from aMessage does not match the name from aMessage2. aOffset is updated.
kErrorParseName in aMessage could not be parsed (invalid format).
