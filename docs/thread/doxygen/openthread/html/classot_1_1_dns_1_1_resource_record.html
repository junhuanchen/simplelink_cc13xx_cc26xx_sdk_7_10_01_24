This template static method searches in a message to find the i-th occurrence of resource records of specific type with a given record name and if found, reads the record from the message.

This method searches in aMessage starting from aOffset up to maximum of aNumRecords, for the (aIndex+1)th occurrence of a resource record of RecordType with record name aName.

On success (i.e., when a matching record is found and read from the message), aOffset is updated to point to after the last byte read from the message and copied into aRecord. This allows the caller to read any remaining fields in the record data.

Template Parameters

[in,out]aOffsetOn input, the offset in aMessage pointing to the start of the first record. On exit and only if a matching record is found, aOffset is updated to point to the last read byte in the record (allowing caller to read any remaining fields in the record data from the message).
[in]aNumRecordsThe maximum number of records to check (starting from aOffset).
[in]aIndexThe matching record index to find. aIndex value of zero returns the first matching record.
[in]aNameThe record name to match against.
[in]aRecordA reference to a record object to read a matching record into. If a matching record is found, sizeof(RecordType) bytes from aMessage are read and copied into aRecord.

§ ReadRecord() [1/2]

static Error ReadRecord ( const Message &  aMessage,
uint16_t &  aOffset,
RecordType &  aRecord 
)
inlinestatic

This template static method tries to read a resource record of a given type from a message. If the record type does not matches the type, it skips over the record.

This method requires the record name to be already parsed/read from the message. On input, aOffset should point to the start of the ResourceRecord fields (type, class, TTL, data length) in aMessage.

This method verifies that the record is well-formed in the message. It then reads the record type and compares it with RecordType::kType and ensures that the record size is at least sizeof(RecordType). If it all matches, the record is read into aRecord.

On success (i.e., when a matching record is read from the message), the aOffset is updated to point to after the last byte read from the message and copied into aRecord and not necessarily the end of the record. Depending on the RecordType format, there may still be more data bytes left in the record to be read. For example, when reading a SRV record using SrvRecord type, aOffset would point to after the last field in SrvRecord which is the start of "target host domain name" field.

Template Parameters
RecordTypeThe resource record type (i.e., a sub-class of ResourceRecord).
Parameters
[in]aMessageThe message from which to read the record.
[in,out]aOffsetOn input, the offset in aMessage pointing to the byte after the record name. On exit, if a matching record is read, aOffset is updated to point to the last read byte in the record. If a matching record could not be read, aOffset is updated to point to the byte after the entire record (skipping over the record).
[out]aRecordA reference to a record to read a matching record into. If a matching record is found, sizeof(RecordType) bytes from aMessage are read and copied into aRecord.
Return values
kErrorNoneA matching record was read successfully. aOffset, and aRecord are updated.
kErrorNotFoundA matching record could not be found. aOffset is updated.
kErrorParseCould not parse records from aMessage (e.g., ran out of bytes in aMessage).
