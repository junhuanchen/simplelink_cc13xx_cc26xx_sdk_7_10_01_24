Detailed Description

This class defines a frame write pointer used by Hdlc::Encoder or Hdlc::Decoder.

This class defines the minimum set of APIs used by Encoder/Decoder for writing an encoded/decoded frame. It is simply a wrapper over a pointer into a buffer indicating where next byte should be written. Along with a write pointer, this class stores a remaining length variable indicating number of remaining bytes that can be written into the buffer.

Note
This class does NOT define the underlying buffer space or how it is being managed.

Encoder or Decoder users are expected to use sub-classes of this class adding the buffer space and implementing the frame buffer management scheme.

Two template sub-class FrameBuffer and MultiFrameBuffer are defined which respectively allow storing a single frame or multiple frames (FIFO queue of frame) in a buffer of a given size.

void UndoLastWrites ( uint16_t  aUndoLength)
inline

This method undoes the last aUndoLength writes, removing them from frame.

Note
Caller should ensure that aUndoLength is less than or equal to the number of previously written bytes into the frame. This method does not perform any checks and its behavior is undefined if aUndoLength is larger than the number of bytes previously written into the frame.
